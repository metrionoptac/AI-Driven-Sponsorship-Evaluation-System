Web Form vs Email: Two Different Paths

Web form fields are pre-structured — the applicant types into named inputs. No LLM extraction needed for form fields. LLM is only used when an attachment is uploaded.

Email Path

Raw text → LLM Extraction (Sonnet) → SponsorshipRequest → LLM Validation (Haiku) → Quality Gate
Cost: ~$0.01 per request | 2 LLM calls

Web Form Path

Form fields → Pydantic Validation → SponsorshipRequest → (+ LLM for attachment if uploaded) → Quality Gate
Cost: $0 if no attachment | 0-1 LLM calls

Form enforces Tier 1 at submission

HTML required attribute prevents form submission without these fields. The user physically cannot submit without org name and amount. Unlike email, there is no AWAITING_INFO state for web form — the form itself is the gate.

Attachment Upload Handling

When the applicant uploads an attachment via the web form

The form fields are already structured (Pydantic-validated). The attachment goes through LLM extraction (same IntakeAgent path as email). Then the system merges both sources:

Rule: Form fields take priority. Attachment fills gaps.
merged.amount = form.amount OR attachment.amount
merged.visibility = form.visibility + attachment.visibility (concatenate, don't override — attachment may have richer package details)
merged.additional_context = attachment.additional_context (extra info from the PDF/DOCX)

LLM Involvement by Scenario

  • Form only, no attachment: 0 LLM calls. Pydantic validates. Cost: $0.
  • Form + PDF/DOCX attachment: 1 LLM call (Sonnet extracts from attachment). Cost: ~$0.005. Merge with form data.
  • Form + image attachment (scan/photo): 1 LLM call + OCR preprocessing. Cost: ~$0.005.
  • All paths: Quality gate runs after merge. If form fields covered Tier 1+2, gate passes regardless of attachment quality.
