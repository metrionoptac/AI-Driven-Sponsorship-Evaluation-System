TIER 1 — Pipeline Blockers ANY 1 MISSING = AWAITING_INFO
FieldGerman (Laura)Why It Blocks the PipelineLLM Validates
organization_name Veranstalter / Antragsteller Eligibility can't check history, blocked orgs. Research can't web-search. Letter can't address. "Who are you?" Is this a real org name? Not a person name, not a generic phrase?
requested_amount Angefragter Betrag / Preisrahmen Budget check impossible. Cost-effectiveness impossible. Research picks wrong depth (QUICK/STANDARD/DEEP). Recommendation can't set amount. "How much do you want?" Is the amount plausible for the stated purpose? (750 EUR for a festival = OK. 50 EUR for stadium naming = suspicious.)

When Tier 1 field is missing or invalid

Pipeline STOPS immediately. State set to AWAITING_INFO. Completeness follow-up email sent to applicant listing all missing Tier 1 + Tier 2 fields. Max 2 follow-ups, then route to human review.

TIER 2 — Evaluation Blockers ASKED IN FOLLOW-UP EMAIL
FieldGerman (Laura)Impact if MissingLLM Validates
purpose Veranstaltungsname / Anlass Strategic fit scoring (28% weight) can't score without knowing what the event IS Is this a real event/project description? Not just "Sponsoring" or "Unterstuetzung"?
visibility Sponsorenpaket + Gegenleistungen Visibility value scoring (19% weight) unmeasurable. Partnership depth unknown. Are concrete benefits named? "Logo" alone is vague. "Logo on 1000 flyers + banner at entrance" is concrete.
event_date Datum / Zeitraum Eligibility date check skipped. Can't detect past events or scheduling conflicts. Is the date parseable? Is it in the future? Is "Sommer 2026" enough or too vague?
region Ort / Region Regional anchoring unknown. Eligibility region match skipped. Is this a real location? Can regional relevance be assessed?
contact Ansprechpartner + Kontaktdaten Can't communicate decision. (Email channel: sender address is fallback.) Is there a name (not just role)? Is there email or phone?

When Tier 2 fields are missing

Pipeline still proceeds if Tier 1 is complete, but evaluation quality is degraded. Missing Tier 2 fields are included in the follow-up email alongside any Tier 1 gaps. Confidence score reduced. If 3+ Tier 2 fields missing, route to human review.

LLM Validation (Claude Haiku) — Replaces Rule-Based Null Checks

  • Field quality: Is purpose: "Sponsoring" actually useful? (No — too vague, treated as missing.)
  • Amount plausibility: Does 750 EUR make sense for a 4-day festival with live band? Does 50 EUR make sense for stadium naming rights?
  • Contact validity: Is contact.name: "Vorsitzender" a name or a role? (Role — treated as incomplete.)
  • Extraction accuracy: Did the extraction miss something the raw text contains? (e.g., amount in a forwarded section)
  • Overall assessment: Per-field quality score (present / vague / missing) + overall completeness verdict
